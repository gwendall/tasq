# Product concepts

Use these documents to understand Tasq before reading implementation-specific
contracts.

1. [`CURRENT_STATE.md`](CURRENT_STATE.md) — what is implemented and what is
   explicitly not shipped.
2. [`PRODUCT_CONSUMPTION_SPEC.md`](PRODUCT_CONSUMPTION_SPEC.md) — Core, Local,
   Server, and Cloud product shapes and their consumers.
3. [`CONTEXT.md`](CONTEXT.md) — canonical coordination language and avoided
   overloaded terms.
4. [`COMMITMENT_DIMENSIONS.md`](COMMITMENT_DIMENSIONS.md) — the compositional
   primitive pyramid, dimension inventory and admission test.
5. [`ARCHITECTURE.md`](ARCHITECTURE.md) — layers, dependencies, and invariants.
6. [`UNIVERSAL_KERNEL_SPEC.md`](UNIVERSAL_KERNEL_SPEC.md) — the neutral target
   model; it does not override current-state truth.

`PRODUCT_SURFACE_MATRIX.json` is the machine-readable support authority.
`LOCAL_CONSOLE_SPEC.md` and `AGENTIC_PRIMITIVES.md` provide narrower conceptual
views.
