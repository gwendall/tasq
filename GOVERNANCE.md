# Tasq governance

Tasq uses maintainer-led governance while the project has one accountable
maintainer. `@gwendall` is the initial maintainer, release owner and security
owner.

Ordinary work happens through pull requests with required tests. Changes to
persisted data, public compatibility, trust boundaries, licensing, governance
or release authority require an ADR and executable acceptance evidence.

Contributors earn commit or review responsibility through sustained, safe
work. Role changes are proposed in a pull request and recorded here before
they take effect. No task assignment, Tasq actor label or automated-agent claim
grants repository or release authority.

The maintainer is responsible for:

- preserving the universal kernel boundary;
- deciding releases and supported surfaces from evidence;
- coordinating private security reports and advisories;
- documenting conflicts of interest and governance changes;
- ensuring project decisions remain in versioned repository documents.

There is no guaranteed community support SLA. A future multi-maintainer model
must replace this document through an ADR rather than inventing an informal
committee.
