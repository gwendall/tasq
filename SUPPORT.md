# Tasq support

Tasq support claims are machine-readable in `PRODUCT_SURFACE_MATRIX.json`.
Current source behavior includes Core embedding and Tasq Local. The public
alpha may be built from source, but no npm package, downloadable release or
stable compatibility promise exists until TQ-607 records a `go` decision and
TQ-603 and TQ-604 pass.

Use public issues now for reproducible bugs and bounded feature proposals.
Use the private channel in `SECURITY.md` for
vulnerabilities. Questions about an adopter's policy, provider credentials,
workflow runtime or private deployment are not automatically Tasq defects.

There is no guaranteed response SLA. A release is supported only on the exact
runtime/platform combinations declared in its metadata. Compilation on another
target is not a support promise.
